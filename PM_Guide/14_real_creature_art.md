# 角色插畫改用原圖（去白底 + 漂浮動畫）

> 對應使用者要求：插畫要跟提供的參考圖「一模一樣」、要可動畫。
> 分支：`claude-pro/real-creature-art`
> 日期：2026-06-10

---

## 問題與根因

舊的手繪 SVG 插畫（海龜/螃蟹/章魚/鯨魚）使用者覺得很醜。多次嘗試由 AI 手寫
SVG 路徑「臨摹」使用者的精緻參考圖都失敗——手寫只能拼湊圓/橢圓，畫不出參考圖
那種平滑貝茲曲線輪廓，成品劣化嚴重。

**根因：方法錯誤。** 由 AI 手寫座標臨摹點陣圖，本就難以達到像素級保真。
正解是直接用原圖，或以工具自動向量化。本機無向量化/影像工具、npm 安裝在此環境
亦失效，故採「直接用原圖」。

## 方案

- 使用者在 `src/photo/` 提供了 10 隻可愛扁平海洋角色 PNG（自有素材）。
  複製為乾淨 ASCII 檔名至 `src/assets/creatures/`（turtle/crab/octopus/whale/shark/
  jellyfish/seahorse/dolphin/pufferfish/narwhal）。
- 新增 `components/creatures/CreatureImage.tsx`：
  - Vite glob 取得圖片 URL。
  - **瀏覽器端 canvas 去白底**：從圖片四邊以 flood-fill 將近白像素設為透明，
    保留內部白色（眼睛、肚子高光）——零外部工具、離線可用。
  - `motion.img` 漂浮動畫（y 上下、淡入），去背完成前透明避免閃白方塊。
- 4 個舊插畫元件改為 `<CreatureImage name=... />`，保持原 export 名稱，呼叫端零改動。

### 考慮過的方案
- **手寫 SVG 臨摹**：失敗（不夠像、醜）。否決。
- **自動向量化（vtracer/potrace）**：最理想（真曲線 + 可動錨點），但本機無工具、
  無法安裝。列為後續（需工具或網路）。
- **直接用原圖 + canvas 去背** ✅ 選定：保真度 100%（就是原圖），離線可行，
  整隻可做漂浮/彈跳動畫。

## 限制 / 待確認（重要）

- 作者**無法在本 session 視覺驗證**：Preview MCP `EPERM` 壞、widget 無法載入本地檔、
  npm 安裝失效故無法用 Node 產圖回看。`vite build` 已通過。
- **請使用者執行 `npm run dev` 實際查看**。若去白底邊緣有殘留白邊（halo），
  調整 `CreatureImage.tsx` 的 `isNearWhite` 門檻（目前 >238）即可。
- **逐部位錨點動畫**（如海龜揮手）需「自動向量化成 SVG」才能做，屬後續分支
  （與使用者所說「項目1可做分支」一致）。目前是整隻漂浮。

## 如何測試（請 reviewer 執行）

1. `npm run dev` → 各空狀態應顯示對應的可愛角色（海龜/章魚等），透明背景、輕輕漂浮。
2. 確認去背乾淨（無白邊、眼睛/肚子白色保留）。
3. `npx vite build` 已通過。
